The following applies to the JAX-WS standalone bundles.

  • Extract ${PRODUCT_NAME} bundle
  • To install on GlassFish v2
    • If you want to use the ${PRODUCT_NAME} overriding the one with Glassfish v2, Download Metro 2.0.1 bundle from http://metro.java.net and follow the instructions in that bundle
  • To use with GlassFish v3.0.1
    • GlassFish v3.0.1 has ${PRODUCT_NAME}, so there is no installation step.
  • To install on Tomcat
    • Set CATALINA_HOME to your Tomcat installation.
    • Run 'ant install', This essentially copies lib/*.jar from the root of ${PRODUCT_SHORT_NAME} extracted bundle to $CATALINA_HOME/shared/lib
    • If you are running on top of JDK6, run 'ant install-api' or use ant -help for detailed information.

If you intend to only invoke an already existing Web service, then alternatively you can use the wsimport and wsgen scripts in the bin directory of the ${PRODUCT_NAME} bundle.

  • Bug fixes
  • Although JAX-WS customizations are portable across implementations, the names of WSDL and schema files generated are not specified. Therefore each vendor may and most likely will generate different names. Therefore, when switching between implementations of JAX-WS, it may be necessary to modify your customization files to reflect different WSDL and schema file names.
  • This holds true for different releases of the ${PRODUCT_SHORT_NAME}. The ${PRODUCT_SHORT_NAME} may in each release change these file names to resolve bugs. If and when this does occur, a developer would need to change the filenames in their customization files to match the new file names.
  • A bug in Apt ant task makes it not work by default. Workaround is to use fork="true" option in the apt task.
